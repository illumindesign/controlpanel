File Structure / Assets

The foundation of a good application is an organized file structure; As development progresses, you may become compelled to separate the functionality of your project into modules or libraries. This separation eases the complexity of your project as it grows or becomes more advanced. The separation also allows for modular development, which can be useful in a number of ways. This section explains the separation as it applies to the structure of this control panel skeleton.

Images

Location: /assets/img/

This is the main images directory, you may need to add more folders depending on what types of images you are using in your project.

Icons

Location: /assets/img/icons/

This is an example of how to extend the Images folder. Example icons have been added, but you may choose to use your own.

Fonts

Location: /assets/font/

This directory stores the physical files for all of the assorted fonts. Move fonts into this folder as necessary.

CSS

Location: /assets/css/

This folder is for the CSS you add to your project.

Layout Elements

Location: /assets/css/illuminstyle.css

#header {
   This is the very top of the page. 40px tall; it contains the logo, main menu, compact menu and user menu.
}
#adminbody {
   This sits directly below the header and contains the rest of the page. It stretches to fill 100% of the remaining space.
}
#loginform {
   The style attributes for the login form.
}
#logo {
   The style attributes for the logo.
}
#topmenu {
   The style attributes for the main menu.
}
#toolheader {
   When logged in, this is the first element in the #adminbody. Visually, it sits between the #header and the #toolcontainer.
}
#toolcontainer {
   This contains all of the navigation elements and controls of the tools installed in the control panel.
}
#toolnav {
   The main navigation on the side which enables you to navigate the controls of the tools in the control panel.
}
#toolcontent {
   The main controls of your control panel tools.
}
#toolfooter {
   The bottom status bar of the control panel.
}
#tooloptions {
   A semi-transparent area for tool control buttons to reside
}
#toolgrid {
   An opaque area for tool controls and forms to reside.
}
#usermenu {
   An area which houses the #profileimage and shows #usermenu-content on hover.
}
#profileimage {
   Holds and sizes the user's avatar.
}
#usermenu-content {
   A drop-down menu that appears when #usermenu is hovered.
}
#usermenu-options {
   A container for basic user options to reside.
}

Responsive Elements

Location: /assets/css/responsive.css

#topmenu_compact {
   Appears as a button when the window is 760px wide or less.
}
#toolnav {
   Shrinks to 65px when the window is 760px wide or less.
}
#topmenu {
   Disappears to be triggered by #topmenu_compact when the window is 760px wide or less.
}

Visual Effects

Location: /assets/css/effects.css

.stretch {
   When applied to a block element, this class will cause the element to resize smoothly on the event of a total size change request.
}
.slide {
   When applied to a block element, this class will cause the element to resize smoothly on the event of a width change request.
}
.fade {
   Make the element fade out when visibility changes.
}
.blur {
   Apply a blur to the element and all its children.
}

Fonts

Location: /assets/css/fonts.css

You must include this file in order to use the fonts declared within. You can add fonts to your project adding the following code: @font-face {
   font-family: MyFontName;
   src: url(../font/MyFontName.otf);
} Once the font is added, you can add the font to your class with the following code: .myClass {
   font-family: MyFontName;
}

Scroll Bars

Location: /assets/css/scrollbars.css

This file is an example of cusomization of the main scrollbars in your project. You can also specify the scrollbar styles of individual classes or elements.

Javascript

Location: /assets/js/

This folder is for the Javascript classes and libraries. Add Javascript files to this folder as necessary.

Functions

Location: /assets/js/functions.js

This is an example of how to extend the Javascript folder. This file contains random functions.

PHP

Location: /assets/php/

This folder is for all of the server-side code.

Globals

Location: /assets/php/global.php

This is the code that is always run before each page load. For the most part, it sets default variables, includes other files.

Pages

Location: /assets/php/pages/

This is the directory where you add new pages to the control panel (eg. /assets/php/pages/example.php). You can link to your new page like this: <a href="index.php?p=example">My New Page</a>

Page Actions

Location: /assets/php/act.php

Actions that can be passed to your page through a link in the address bar. <a href="index.php?p=example&act=doThis">Action on My New Page</a>

Form Submission

Location: /assets/php/post.php

This file handles all of the form submissions.

Functions

Location: /assets/php/act.php

Add global functions here.

Classes

Location: /assets/php/classes.php

Put classes here.. if you have many classes, you may want to create a "classes" subfolder instead.

Databases

Location: /assets/php/pdo.php

This enables the control panel to utilize a database connection through PDO. To enable databases, add your information to this file and uncomment the following code to global.php (line 15) require_once('assets/php/pdo.php'); To perform a database query, use code the $bbcp_connection variable: try {
   $q = $bbcp_connection->query("SELECT * FROM `my_settings` LIMIT 1");
   $settings = $q->fetch();
} catch (PDOException $E) {
   die('Query Error: '.$E);
} To read more about PDO, click here.
